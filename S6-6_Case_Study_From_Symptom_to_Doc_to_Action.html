Case Study: From Symptom to Doc to Action
Section 6
Scenario: An RK24 is in operation. The operator notices the hydraulic system feels sluggish under load. Hydraulic oil temp is in the upper amber band. The CR1081 display shows a fault code. The operator brings the machine to a safe stop.
1
Operator: Observe, Stop, Secure
Sluggish hydraulics + amber temp + display fault = controlled stop required. Lower attachment, apply park brake, record fault code from CR1081 display before shutdown.

Open IFM CR0020 Software Manual (Doc 07e) — look up fault code. Code maps to hydraulic pressure sensor input out of range. Cross-check with CR1081 System Manual (Doc 07b) for display alarm context.

Why the Document Chain Matters
In this scenario, four separate OEM documents were needed to trace a single symptom to a root cause: CR0020 Software Manual → CR1081 System Manual → CR0020 System Manual → Rexroth pump spec. Each document answered one question. Without the chain, the technician would be guessing — and replacing parts speculatively.
Outcomes From Good Documentation
Root cause identified first — no parts replaced unnecessarily
Repair time reduced — technician follows a structured diagnostic path
Service record cites the exact doc sections used — supports warranty claim if applicable
Next technician can review the service record and understand the full diagnostic path
If the same symptom recurs — the service record shows what was already ruled out
The Operator's Role in This Chain
The operator's fault code note and shift log entry are step zero in this chain. Without an accurate fault code, the technician starts at square one. The operator's observation of "sluggish under load" and the temp band narrows the diagnostic field before the technician opens a single document.
Every symptom has a document path. The skill is knowing which document to open first — and that skill comes from understanding the subsystem map in this section.
